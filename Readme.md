# Snake/Coil-in-the-Box — 3D Web Viewer

A small React Router + react-three-fiber app that visualizes results from the
`snake-in-the-box` optimization scripts (SCIP / Gurobi / Xpress) as an
interactive 3D grid. Lets you pick a grid size `n` and an algorithm, then
step through the resulting path/cycle one vertex at a time.

## Why static JSON instead of a live backend

SCIP (and the other solvers used here) has no built-in way to expose a
running solve over HTTP — there's no server component to call from the
browser. So the workflow is split in two:

1. **Offline**: run the solver scripts in this repo (`snake/`, `coil/`,
   `convert_me/`) as usual, however long that takes.
2. **Export**: dump the resulting vertex sequence to a small JSON file.
3. **Online**: the web app just reads that static JSON — no solver, no
   server-side computation, no long-running requests.

This also means the site can be hosted anywhere that serves static files
(Vercel, GitHub Pages, etc.) with zero backend cost.

## Where the results live

Each `(algorithm, n)` pair is one JSON file under
[`app/data/`](./app/data):

- [`app/data/snake_n2.json`](./web/app/data/snake_n2.json) … `snake_n6.json` —
  open-path (snake) results, one file per grid size `n = 2..6`
- [`app/data/coil_n2.json`](./web/app/data/coil_n2.json) … `coil_n6.json` —
  closed-loop (coil) results, same `n` range
- [`app/data/algorithms.ts`](./web/app/data/algorithms.ts) — metadata for the
  algorithm dropdown (label + which script in the parent repo produced it)

**Note:** the JSON files currently in the repo are placeholder data
(`is_dummy: true` in each file), generated by a simple grid-filling walk —
not real SCIP/Gurobi/Xpress output. To show an actual solve result, replace
the `path` array in the matching file with the real vertex sequence and set
`is_dummy: false`.

JSON schema (see [`app/types/snake.ts`](./web/app/types/snake.ts)):

```jsonc
{
  "d": 3,                 // dimension, fixed at 3
  "n": 4,                 // points per edge (grid is n x n x n)
  "type": "snake",        // "snake" (open path) | "coil" (closed loop)
  "is_cycle": false,      // true for coil results
  "path": [[0,0,0], ...], // ordered list of visited grid coordinates
  "is_dummy": true,
  "is_verified_optimal": false,
  "note": "human-readable description"
}
```

Sample code for converting result to json : [`export to json`](./export_to_json.py)

## Why this is a separate GitHub repo, not on ZIB's GitLab

`git.zib.de` is behind the ZIB network/VPN, and Vercel's Git integration
only connects to GitHub, GitLab.com, or Bitbucket — not self-hosted GitLab
instances it can't reach. Standing up a CI pipeline to bridge that would
have been more setup than the viewer itself needs, so the web app lives in
its own public GitHub repo and deploys straight to Vercel from there. The
solver code and research data stay on `git.zib.de` as usual; only the
static JSON exports and the viewer code are mirrored here.

## Running it locally (no website needed)

If you don't need the hosted version and just want to click through
results yourself:

```bash
git clone <this-repo-url>
cd web
npm i
npm run build
npm run dev
```

Then open `http://localhost:5173`.

⚠️ **Caching gotcha:** the JSON files are bundled into the app at build
time (via Vite's `import.meta.glob`), not fetched at runtime. If you edit a
JSON file and the change doesn't show up, it's almost always Vite serving a
cached module — stop the dev server (`Ctrl+C`) and run `npm run dev` again.
A hard browser refresh alone usually isn't enough.

## Architecture

```
snake-in-the-box/            (ZIB GitLab — solver code, unchanged)
├── snake/ coil/ convert_me/ xpress/   <- SCIP / Gurobi / Xpress solve scripts
└── web/                      (this repo — GitHub, deployed to Vercel)
    ├── app/
    │   ├── data/              <- JSON results + algorithms.ts metadata
    │   ├── types/snake.ts     <- shared TS types for a result file
    │   ├── components/
    │   │   ├── SnakeCubeView.tsx  <- react-three-fiber 3D grid + tube renderer
    │   │   └── ClientOnly.tsx     <- guards Canvas from SSR (no window/document)
    │   ├── routes/home.tsx    <- n selector, algorithm dropdown, playback controls
    │   └── styles/            <- plain CSS (Home.css, SnakeCubeView.css)
    ├── vite.config.ts         <- Tailwind + React Router plugins
    └── react-router.config.ts <- Vercel deploy preset
```

Data flow: **solver script -> JSON export -> `app/data/*.json` -> bundled at
build time -> rendered client-side as a 3D grid with the visited path drawn
as a tube, revealed step by step.**
